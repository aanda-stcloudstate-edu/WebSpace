Introduction:

This lab exercises the ability to:
  • perform I/O viaONCHANGE text boxes in the context of event driven programming
  • use the two division operators to implement conversion between binary and decimal in both directions
  • use conditional logic
  • use functions

Lab Assignment:

Create a new folder, lab9, and move to it.

Create a new Webpage, BinaryDecimalConverter.html, following these instructions:

Title the page appropriately.

Create two appropriately labeled textboxes on your Webpage.

Use the ONCHANGE attribute to convert the binary number in one box to the decimal in the other, and vice versa.

The binary number will have to be stored as a decimal whose digits are constrained to the values of zero and one.
E.g. binary five [ 101 ] is stored as decimal one hundred one.

The data is constrained to one byte of storage so that the maximum value is a positive decimal integer between zero and 255 inclusive,
and the maximum binary representation is 11111111.
Therefore, use alerts to complain to the user that their input data is outside those ranges.
(Because we have restricted the number of binary digits to a maximum of eight, there is no need to use a repetition statement.)

Use another alert to indicate that a value containing an illegal digit ( [2 .. 9] ) was entered into the binary number field

Label the text boxes appropriately, i.e. each textbox should be labeled with the base and valid input range.
E.g. Base 10 [0 .. 255]

Ensure that in your code identifiers (e.g. variable and function names) are labeled appropriately.

You should use a separate function for base conversion in each direction respectively.

Extra Credit: use the JavaScript switch statement to convert either the decimal or binary value additionally to and from hexadecimal via two single-character text boxes.
