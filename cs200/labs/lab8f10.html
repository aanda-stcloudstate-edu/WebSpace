Introduction:

This lab exercises the ability to:
  • perform I/O via ONCHANGE text boxes in the context of event driven programming
  • use the two division operators, "/" & "%" to implement conversion between binary and decimal in both directions
  • use conditional logic to validate input
  • define and use functions

Lab Assignment:

  • Create a new directory, Lab8, in your class account directory.
  • Move into your Lab8 directory
  • Create the file BinaryDecimalConverter.html
  • Use HTML comments at the top of this file identifying:
    • where the file is located (e.g. ~/cs200999/labs/Lab8/BinaryDecimalConverter.html)
    • the author (you)
    • what the program does
    • when you created the file
    • when you last modified the file
  • When implementing your program, as often as is practical, repeat the following steps:
    1. modify your Web page file via the editor
    2. save the file
    3. refresh the browser open to your Web page
    4. verify that your modifications did what you've intended
  • Title your Webpage appropriately.
  • Create two appropriately labeled textboxes on your Webpage.
  • Use the ONCHANGE attribute to convert the binary number in one box to the decimal in the other, and vice versa.
    • The binary number will have to be stored and represented as a decimal whose digits are constrained to the values of zero and one.
      E.g. binary five [ 101 ] is stored as decimal one hundred one.
    • The numeric value is constrained to one byte of storage so that the maximum is a non-negative decimal integer between zero and 255 inclusive [0 .. 255].
    • The maximum binary representation is therefore 11111111.
      Therefore, use the alert() function to complain to the user that their input data is outside the specified ranges.
      (Because we have restricted the number of binary digits to a maximum of eight, there is no need to use a repetition statement.)
  • Use another alert() to indicate that a value containing an illegal digit ( [2 .. 9] ) was entered into the binary number field
  • Label the text boxes appropriately, i.e. each textbox should be labeled with its respective base and valid input range.
    E.g. Base 10 [0 .. 255]
  • Ensure that identifiers in your code (e.g. variable and function names) are labeled appropriately.
  • Use a separate function for base conversion in each direction respectively.
  • Extra Credit: use the JavaScript switch statement to convert either the decimal or binary value additionally to and from hexadecimal via two single-character text boxes.
  • Test your code using either the conversion tool on the Data Representation Page from Reed's resource page or use the conversion tool on this page.
  • When finished, submit your Lab 8 solution Webpage to the appropriate dropbox.
