Homework Assignment 1: DESIGNING and DESCRIBING ALGORITHMS

Definition: algorithm: A sequence of well-defined instructions for carrying out an operation or solving a problem.

Definition: loop: A sequence of instructions that are repeated until some condition is met, or indefinitely.

For each of the following problems you will solve:

  1. describe your algorithm using pseudocode (use Program 1-4 of Forouzan as a template)
  2. describe your algorithm using a flowchart (see Figure 1-13 and Appendix C)
  3. Demonstrate that your algorithm works properly by describing what happens at each step, from start to finish, for a good (non-trivial) test case.

Do not write code in any programming language,

Be neat.

Caution: each variable (e.g. bath_length) that you use in an expression (e.g. bath_length * bath_width), a comparison (e.g. if (time == 60) then), a return value, or a write value, must have been initialized with some actual value either by an assignment (e.g. time = 0) or by a read (e.g. read bath_length).

  1. Describe an algorithm that takes as input a positive integer N and returns the sum of the first N squares:
    12 + 22 + 32 + ... + N2

    Note that this algorithm will contain a loop.

  2. On many bank accounts, interest is compounded daily. For example, if the nominal yearly interest rate is 6% = 0.06, the daily rate is (0.06/365). Thus, an initial balance of $1,000.00 earns $1,000.00 * (0.06/365) = $0.16 at the end of its first day on deposit; the new balance is $1000.16.

    On the second day, the interest will be (0.06/365) times $1000.16, which will be added to the account, and so on.

    Describe an algorithm which takes as input the following three values,

    • an initial balance,
    • an interest rate, and
    • the number of compounding days,
    and produces as output the final balance after the specified duration. This algorithm, too, will require a loop.

  3. Describe an algorithm which, given any amount, from $0.00 to $1.00, will be able to make change from a dollar using the fewest number of coins. (e.g. given $0.08, it will print out the value of $0.92 as: 3 quarter(s), 1 dime(s), 1 nickel(s), and 2 cent(s).)
