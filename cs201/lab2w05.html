Useful tools:

  • touch: "touch foo" creates the empty file foo.
  • screen:  this command creates multiple virtual windows in your telnet window.  The following commands should be sufficient:
    • invoke screen at the prompt.
    • ^a c : creates another screen and switches to it.
    • ^a [spacebar] : toggles to the next existing virtual screen (that you had created at some time previously using ^a c).
    • exit : destroys the current virtual screen.
  • Wildcards: [see Ch. 3, pages 66-68, in the Unix text] special characters that the shell uses to search for classes of filenames
    • "?": replace with any one text character.
    • "*": replace with zero or more arbitrary characters.
    • "[...]": replace with any one of the characters enclosed in the brackets.
      • A range can be indicated: e.g. [A-Z] means any upper case alphabetic character.
  • Pipes: [see Ch. 5, pages 151-153, in the Unix text] a pipe, indicated by the vertical bar character "|",  sends the output of the preceding program in the the input of the successive program.
    • e.g. "prog1 | prog2" has prog2 processing the output of prog1
