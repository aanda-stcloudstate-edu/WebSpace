Introduction:

This lab provides experience with arrays and iteration. The task is to compute a sequence of prime numbers via the Sieve of Eratosthenes.

Lab Assignment:

Problem Motivation and Definition:

Public key cryptography is based on prime numbers and the difficulty of factoring large compound numbers into primes.
The Sieve of Eratosthenes is an ancient algorithm for sequentially finding primes.
This algorithm will require arrays and iteration through those arrays.

Solution Specification:

    1. Create a new directory, lab6, and move to it.
    2. Create a Sieve_of_Eratosthenes function which accepts two integer arrays.
      One array will be populated by the function with the contiguous sequence of integers starting with 2 up to a limiting value (passed into the function).
      The other array will be a sequential list of primes starting with 2. The number of primes found will be passed out of the list (it is also the number of array elements used in this array).
      Implement in this function the following Sieve of Eratosthenes algorithm.

      Note: to find all the primes up to N, it is only necessary to eliminate prime factors up to sqrt(N). (can you deduce why this is so?)

    3. Write a program which prompts the user for a positive integer (>1) to search for primes up to (limited by the static size of the arrays).
      The program will then print, with message, the number of primes found, followed on the next lines by the set of consecutive primes, ten per line.
      Be sure to avoid using magic numbers in declaring your arrays.
    4. Paste the list of the first thousand prime numbers into a file.
      See Chapter 12 for information on how to read the contents of this file into your program.
      Compare up to the first thousand primes found by your function with those from the reference list in the file.
      Inform the user whether their list of primes is OK up to the first 1000.
    5. When correct, generate via enscript an HTML printout of the source, and demonstrate, via script, the source and the running of the program data.
      Submit to D2L.
