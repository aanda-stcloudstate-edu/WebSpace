CSCI 201

Lab 2

Introduction:

This lab examines and lists the local integer and floating point environments with respect to limits on csci (a Sparc architecture) and compares them with those on an Intel architecture.

Lab Assignment:

  1. Create a new directory, lab2, and move to it.
    1. run: cp ~aanda/cs201/source/CHAppndx/PE-0[12].c++ . {dot}
    2. run: cp ~aanda/cs201/source/CHAppndx/PE-0[12].Intel . {dot}
    3. compile: g++ PE-01.c++ -o PE-01.exe
    4. compile: g++ PE-02.c++ -o PE-02.exe
    6. Run: each executable {*.exe}
      7. saving the output in PE-01.Sparc and PE-02.Sparc respectively.
    7. Run diff: on each *.Sparc and *.Intel file respectively and save each output to a new file with a *.diff suffix
    8. Edit each .diff file to add your comments on what the diff output means and implies about the numeric limits on each architecture.
    9. Print a copy of the *.Sparc files together using lpr outputfilename.
    10. Submit one copy of the *.Sparc and the *diff files to D2L.
    11. Retain printout for your notes and exams.
  2. Write a program to test the integer division equation:
    a == (a/b)*b + a%b
    for all signs.
    1. Prompt the user for both values.
    2. Show that the relation holds.
    3. Submit to D2L the program source and a typescript file showing the program run for a variety of input pairs including pairs having all possible signs.
  3. limits.h and float.h have been superceded by the new <limits> library file
    1. run: cp ~aanda/cs201/local_limits.cpp .
    2. Compile and run this file storing the output in local_limits.Sparc
    3. Submit a copy of local_limits.Sparc to D2L
    4. Print a copy each of local_limits.cpp and local_limits.Sparc for yourself using lpr outputfilename.

Editing Suggestions:

  • run vim
  • set the following flags:
  • ":set cin sw=4" {this maintains indenting.}
  • ":syntax enable" {provides automatic syntax highlighting to help illuminate structure and mistakes}
  • "yank" and "put" similar lines to save editing work {yy and p commands}
  • use two terminal windows Write from the editor periodically {":w"} while compiling and running in the other window to catch syntax and other errors early.
