Useful tools:

  • touch: "touch foo" creates the empty file foo.
  • Wildcards: special characters that the shell uses to search for classes of filenames
    • "?": replace with any one text character.
    • "*": replace with zero or more arbitrary characters.
    • "[...]": replace with any one of the characters enclosed in the brackets.
      • A range can be indicated: e.g. [A-Z] means any upper case alphabetic character.
      • Except for ranges, the order of the characters is ignored.
      • There are no separators between characters, e.g., [a,b] matches the three characters, "a", "b", and ","(comma).
  • Pipes: a pipe, indicated by the vertical bar character "|",  sends the output of the preceding program in the the input of the successive program.
    • e.g. "prog1 | prog2" has prog2 processing the output of prog1
