Debugging Hints & Suggestions

Development

  • Use lots of white space and consistent structural indentation to maximise the readability of your program.  The easier it is to read, the easier it is to debug;
  • Consider starting with the psuedocode version of your program as a basis for the descriptive comments in your program.  Insert code below the comments which describe it;
  • Solve the smallest possible version of the problem you're attacking.  Once that is correct, scale up;
  • Procede incrementally.  Ensure you're program is correct in stages:
    • code
    • compile
    • test
    • repeat
  • Test with the most diverse possible set of test data at each stage;
  • Use an editor with syntax highlighting such as vim or emacs;
  • Follow the suggestions in  the SOFTWARE ENGINEERING AND PROGRAMMING STYLE section of each of the chapters we have or are covering;
  • Follow the suggestions and directives in Appendix H, Program Standards and Styles;

Syntax errors

  • If the compiler generates a list of error messages, look for a problem at location of the earliest line number mentioned, and the lines preceding  it.
    • After you think you've corrected it, recompile.  Often, many of the subsequent error messages were propogated from the one you corrected and will disappear.
  • Make sure:
    • you have followed all directives in the TIPS AND COMMON PROGRAMMING ERRORS section of each of the chapters we have or are covering;
    • all variable are declared, and also initialized when necessary;
    • all statements end with a semicolon;
    • all delimiters (e.g. parentheses, apostrophes, quotes, brackets, and braces) are matched;
      • To this end, I recommend always starting a delimiter bracketed expression with the matched delimiters and then inserting into them.  (e.g., begin writing  a string literal with "", and then insert into the pair)
    • all necessary libraries have been properly included;

Logic errors

  • Trace by hand: try to recreate by hand what you expect each successive statement will do.
  • Program tracing: print out the values of variables at intermediate stages to track how they are changing.  Are the changes what you expected?
