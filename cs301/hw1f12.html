Programming Assignment 1

Dealing Sorted Playing Card Hands

Write a program to generate and print a set of sorted playing card hands from a single deck.

Specifications:

  1. The program will read, in any order, the following program flags and options from the command line:
    • number of hands (ordered sets of cards) [default == 4]
    • number of cards per hand [default == 5]
    • sort first on value or suit? [default == value]
    • is Ace high or low in sort? [default == high]
    • suit ranking order from high to low [default == "shcd"]
    • maximum number of cards/hand to discard and redraw from deck [default == 0]
    • srand() argument [default == srand() uses time() as an argument]
    • -d "d" stands for dump or debug. You will output intermediate values to help in debugging.
      • feel free to add a facility for this option to accept an integer argument to control the amount and degree of debugging information
    • Add any other options you'd like.
    • Either of the following two options will prevent the normal execution of the program. All other options will be ignored
    • -V output the version ID. (this will help you identify which executable you're using)
    • Calling the program with NO arguments will be treated as a cry for help. In this case, output usage instructions.
  2. The program will output to the standard output stream for each hand:
    1. An ordinal identifier for that hand
    2. The cards in the hand ordered from low to high on its own line.
      For each card, the value will precede the suit. (See Programming Problem 1.4 for an example)
  3. After all of the hands have been displayed:
    1. The program will output the number of cards remaining in the deck.
    2. If cards can be discarded and redrawn, the program will prompt the user for the number of cards they wish to discard/redraw.
      The program will redisplay the hand with each card's ordinal position displayed underneath it.
      The program will prompt the user for each position to discard/redraw.
    3. After all of the hands have been updated, they will be redisplayed, each hand in sorted order.
    4. The program will output the the revised number of cards remaining in the deck.
  4. The method to select the cards for a hand should use an array of pointers to a static array of 52 ordinal values.
    Once a card has been selected, it must not appear in another hand or the same hand, before or after discard/redraw.
  5. The method to sort the cards in a hand should use that same array of pointers to a static array of 52 ordinal values.
  6. Note: In both the selection and sorting methods, only the pointers should be rearranged.
  7. Ensure that your program detects bad, undefined, and out-of-range input errors both on the command line and during execution. Error messages should go to standard error.
    • A bad command line argument should prevent the normal execution of the program. Output usage instructions (as in the case with no arguments).
    • Ensure that your test data exercises these error handlers.
  8. Focus on the modularity of your design. Try to maximize cohesion and minimize coupling.
  9. Before the program exits, be sure to release back to the heap all dynamic memory objects.
