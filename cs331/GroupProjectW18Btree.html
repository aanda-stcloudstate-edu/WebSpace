B+Tree Group Project

Your team has previously implemented Component A:

Component A

The generation of a sequence set of records
Component A lays the foundation for the second team component which implements:

Component B

The creation of a B+tree index of that sequence set.
You will implement a B+tree having the following attributes:
  • all options should be available through the command line
  • use a configurable order B+tree
    • consider using a lower order (<=10) for testing and initial debugging
  • records will reside in the leaves only in a seqence of linked blocks. Higher order nodes will contain only keys.
    (We are borrowing the idea of linked blocked records from Chapter 10.1 & 10.2, and introduced in the first paragraph of p. 401 in Section 9.9, and defined in Section 9.10)
    • We can simplify the avail list by using identically-sized blocks for both the sequence set blocks and the tree node blocks.
  • Essentially, we are implimenting Programming Exercises 10.19, 10.21, and 10.22, and optionally, 10.20
Write a test driver that demonstrates that your B+tree works.
At a minimum your program should be able to (via menu):
  • open the files of records along with their B+tree indices
  • print all the information about a record
  • modify fields in any record
  • insert records
  • delete records
  • verify that the B+Tree file is consistent, but rebuild if inconsistent
  • rebuild the B+Tree
  • any other useful operations you think of that you can implement and test

You should have a method of printing (dumping) the intermediate contents of the B+tree so that you can determine whether your implementations for building, insertion and deletion are functioning properly. This dump should be sufficiently readable to help facilitate debugging.
