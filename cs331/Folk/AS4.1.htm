Fields and Records

Classes for buffers of records

 

A stream:  <<  ( in C++)

Just put the characters to the file designated. 

There is no way to get it apart once the data is in the stream

We loose the fundamental pieces

Fields:  The smallest meaningful unit of information in a file

So how do we change the way we view data or write it out so we can get it back?

This is the question we need to answer in this class.

 

There are 4 basic methods for organizing fields so we can retrieve the data.

1.       Each field is given a fixed length

Ø    We have to pad (with blanks) when the contents does not need all the space

Ø    We have to truncate when the data is too long for the space.

Ø    This is the easiest technique so it is appropriate to use if the data has a natural size.

 

 

 

2.     Each field begins with a length indicator (Length-Based)

 

Ø    If we are lucky we can store the length in 2 bytes (256)

Ø    It takes time to calculate the lengths

Ø    This works well for data that has a natural wide variation in size

 

 

 

 

3.     Separate the fields with delimiters

Ø    the delimiter cannot be something that would occur in the data

Ø    white-space, blank, newline, tab can often be used in character data

Ø    but cant be used in addresses

 

 

 

 

 

4.     Use Keyword = Value expressions

Ø    Still requires a delimiter but the fields can be written in any order

Ø    Takes a lot of space

Ø    looking at a dump will tell you what is in the file

Ø    programs dont have to know how the data is stored

 

 

 

Records:  Collection of fields that we want, sometimes, to treat as an aggregate.

Records can be fixed in length or variable in length.

 

How do we organize the records so that we know when one record stops and the next begins?

Ø    Fixed Length Records

o      Requires the records be a fixed number of bytes in length

o      Most common because they are the easiest

o      Within each record we can use any of the field storage techniques

 

 

 

Use fixed length fields

Use delimited fields

Use field counts

Ø    Variable length record fixed number of fields

uses delimiters or counts to distinguish the fields

 

 

 

 

Ø    Each record has a length indicator

can use counts or delimiters to distinguish the fields

you dont need a fixed number of fields

 

 

 

 

 

 

Ø    Use an index to keep track of the start of each record

You can use this to figure out how long the record is

Fields use delimiters or counts

Ø    End each record with a delimiter
